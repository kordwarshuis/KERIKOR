## RSA and public / private key pairs


:::note

Source: Grok

:::

**RSA** is one of the first and most widely used **public-key cryptosystems** (also called asymmetric cryptography). It directly embodies the concept of **public and private key pairs**.

### Core Relationship

- **RSA generates and uses a mathematically linked pair of keys**:
  - **Public key** — Can be freely shared with anyone.
  - **Private key** — Must be kept secret by the owner.

- The public key is used for **encryption** (or signature verification).
- The private key is used for **decryption** (or signing).

This is the opposite of symmetric cryptography (like AES), where the same secret key is used for both encryption and decryption.

### How the Keys Are Generated in RSA

1. Pick two large secret prime numbers *p* and *q* (kept hidden).
2. Compute the modulus *n = p × q* (this becomes part of both keys).
3. Compute Euler's totient φ(n) = (*p*-1)(*q*-1).
4. Choose a public exponent *e* (commonly 65,537) that is coprime with φ(n).
5. Compute the private exponent *d* such that *d × e ≡ 1 (mod φ(n))* (i.e., *d* is the modular inverse of *e*).

**Resulting keys**:
- **Public key** = (*n*, *e*)
- **Private key** = (*n*, *d*)  or simply *d* (since *n* is public)

### How the Relationship Works in Practice

**Encryption / Decryption**:
- Anyone can encrypt a message using your **public key**:  
  **ciphertext = plaintext^e mod n**
- Only you (with the **private key**) can decrypt it:  
  **plaintext = ciphertext^d mod n**

**Digital Signatures** (the reverse direction):
- You sign a message with your **private key**.
- Anyone can verify the signature using your **public key**.

The mathematics (Euler's theorem + modular exponentiation) guarantees that applying the public operation and then the private operation (or vice versa) recovers the original value — but doing the reverse without the private key is believed to be computationally infeasible.

### Why It Works (Security Basis)

The security of RSA rests on the **integer factorization problem**:
- It is easy to multiply two large primes to get *n*.
- It is extremely hard (with current technology) to factor *n* back into *p* and *q* when *n* is very large (e.g., 2048 bits or more).
- Knowing *p* and *q* allows you to compute φ(n) and then *d* from *e*. Without them, you cannot efficiently find *d*.

### Summary of the Relationship

| Aspect            | Public Key (*n*, *e*)            | Private Key (*n*, *d*)           |
| ----------------- | -------------------------------- | -------------------------------- |
| Distribution      | Shared openly                    | Kept secret                      |
| Primary use       | Encrypt data / Verify signatures | Decrypt data / Create signatures |
| Mathematical link | *e* is inverse of *d* mod φ(n)   | *d* is inverse of *e* mod φ(n)   |
| Derived from      | Chosen *e* + primes              | Computed from *e*, *p*, *q*      |

**In short**: RSA *is* the algorithm that creates and exploits a matched public/private key pair based on number theory, enabling secure communication and digital signatures without needing to share secret keys in advance. It is the foundational example of asymmetric cryptography.
